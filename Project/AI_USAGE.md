# AI Usage (CS 348 Semester Project)

This section explains how AI tools were used in this project.

##Tools Used
Cursor, which is an IDE with an AI coding assistant
General LLM chat tools(chatGPT) were used sometimes for simple explanations
Most of the AI-assisted work was done in Cursor

##What AI Helped With
AI helped with the following tasks:
Thinking about the database schema and SQL migration ideas
Organizing the Flask API structure
Writing and checking parameterized queries
Connecting Flutter screens, dialogs, navigation, and API calls
Debugging runtime errors, such as layout problems, async issues, and controller disposal
Drafting project documentation and demo outlines

##How I Checked and Modified the AI Output
All AI-assisted work was reviewed and tested before submission.
I ran all SQL code in MySQL and checked the results with real queries.
I checked that the schema matches backend/database/schema.sql.
I checked that API responses matched the database results.
User input values are passed as parameters using %s.
User input is not directly joined into SQL strings.
I checked the project against the course requirements
Requirement 1: Events CRUD
Requirement 2: Reports and filters
Stage 3 topics: SQL injection, indexes, and transactions
I edited AI-generated code to match my project style, naming, and grading requirements.
